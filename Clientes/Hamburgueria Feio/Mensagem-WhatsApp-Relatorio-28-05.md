## 📌 Notas de uso

| Versão | Quando usar |
|---|---|
| **Completa** | Primeiro envio · grupo formal · contexto completo |
| **Intermediária** | Comunicação recorrente · cliente já familiarizado com o formato |
| **Resumida** | Grupos dinâmicos · cliente prefere objetividade |
| **Estratégica** | Justificativa de continuidade/expansão de orçamento · reuniões |

### Boas práticas de envio

- Enviar a mensagem antes do anexo — o texto contextualiza o material
- PNG é recomendado para visualização imediata no grupo (preview embedded)
- PDF como follow-up para arquivamento formal
- Aguardar resposta do cliente antes de iniciar próximos tópicos

---

## 📐 Padrão de linguagem

Todas as versões seguem o padrão profissional documentado em [[project-communication-standard]]:

- ✅ Saudação formal ("Bom dia")
- ✅ Despedida cordial ("À disposição", "Atenciosamente")
- ✅ Dados em negrito (`*texto*`)
- ✅ Estrutura por seções
- ✅ Tom técnico e direto
- ❌ Sem gírias ou expressões coloquiais
